## 📖 前言

这是一本自动化（全英文）的学习指南，在这里你将会理解我们学的这些课程到底怎么用，从而自定义属于自己的学习方向。本指南的核心目标有两个：

1. **提供专业导论级别的 Overview**：帮你建立宏观的知识体系。
2. **沉淀硬核的课程资源**：包括往年题、课程笔记和高价值参考资料。

这里致敬一下csdiy的作者，本网站借鉴了他的风格。
## 🎯 为什么要有这个指南？

首先，如果你初入大学，建议你先读一下*** [《上交生存指南》（上海交通大学生存手册）](https://survivesjtu.gitbook.io/survivesjtumanual/)***，这能让你提前了解大学学习的真相。

如果你是一名刚刚步入大学校园的自动化或相关 EECS 专业的同学，你可能很快就会在庞杂的课表中体会到一种**“迷茫感”**。

**自动化专业的痛点在于：**
课程太多太杂，且长久以来似乎没有“专属于自动化”的对口行业。三年学下来，你可能会觉得你什么都能做，但在求职时却总是要切入别人的基本盘，面临着极其尴尬的竞争：

* ⚡ **电气方向**：电气工程及其自动化的学生比我们更精通强电。
* 💻 **IT 方向**：代码能力和软件架构能力不如纯 CS 大类的学生。
* 🔬 **IC 方向**：集成电路与微电子方向的学生比我们更懂底层芯片物理。
* ⚙️ **控制理论**：就连原本作为核心壁垒的传统控制理论，也逐渐被机器学习和深度学习大举交叉入侵。

这导致我们自动化的学生，**在很多行业没什么绝对竞争力，无法应对岗位的需求，为什么我不直接读上面对应的专业**。但好在，**真正属于自动化专业的增量市场正在爆发**：无人机、自动驾驶、人形机器人等。我们在这些领域拥有极多的选择，正是因为前期积累的宽泛基础知识，赋予了我们无缝切入这种复杂交叉工业体系的能力。

我们学的课程实在**太多、太杂了**。从《高等数学》到《大学物理》，从《电路原理》到《C语言》，再到后来的《自控原理》和《微机原理》…… 很多时候，这些课程在书本上就像是一座座互不相干的“孤岛”。你可能会在深夜算微积分时怀疑人生：“学这些到底有什么用？”

**建立这个指南，就是为了帮你把这些“孤岛”连成一片大陆。以及提前为你可能哪一方面都不精通敲响警钟**

在这里，我们不念枯燥的教学大纲，而是要告诉你两件事：
1. **学以致用**：你现在学的每一门“天书”，在未来具体的工程项目里到底负责解决什么真实问题。
2. **串珠成线**：打破“万金油”的魔咒，帮你尽早看清前方的路，锁定自己的主攻方向。

自动化的魅力在于它的“宽口径”。当你熬过大一、大二的公共基础课后，你会发现你的技能树足以支撑你走向三条截然不同、但都充满机遇的道路：

* 🔌 **EE 路线（偏硬件与底层）**
    深入物理世界的骨干。如果你对电路和硬件感兴趣，你可以钻研**硬件电源设计、电机控制（FOC）、嵌入式系统**，或是投身于极具壁垒的**芯片制造与 IC 设计**。这是一条越老越吃香的硬核工程之路。

* 💻 **CS 路线（偏纯软件）**
    如果你发现自己对敲代码情有独钟，完全可以利用良好的理工基础直接转向纯计算机科学（建议参考开源的 [《CS 自学指南》](https://csdiy.wiki/)），去深耕后端开发、数据库系统或纯软件架构，或者是 AI 方向。

* 🤖 **交叉路线（系统集成与具身智能）**
    **这是自动化专业得天独厚的“甜点区”。** 当你既懂硬件传感器的底层逻辑，又具备强大的 C++ 软件工程能力，同时还掌握控制理论与算法时，你就可以去挑战当代最酷的科技——**无人机、自动驾驶汽车、人形机器人**。在这条路上，你将成为整合软件与物理硬件的“工业大脑”。

无论你最终倾向于哪条路径，本指南都希望能在你的低年级阶段充当一个路标，陪你少走弯路，点亮属于你的技能树，让你少点迷茫，尽早确认自己真正感兴趣的发展方向。避免你走向了没法升学，同时也找不到工作的窘境（或者能让你提前接受）
